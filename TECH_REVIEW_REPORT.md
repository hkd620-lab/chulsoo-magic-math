## 1. [초급] Flip Master: 스피드 퀴즈 시스템
**목표**: 역수의 형태(자연수 ↔ 단위분수, 진분수 ↔ 가분수)를 직관적이고 빠르게 인지하는 '기초 체력' 기르기.

### 1-1. 데이터 구조 설계 (Data Structure)
스피드 퀴즈 특성상 지연 없는 화면 전환과 랜덤한 문제 출제가 필수적입니다.
```typescript
interface QuizItem {
  id: string;
  questionType: 'natural_to_fraction' | 'fraction_to_fraction'; // 자연수↔분수, 분수↔분수
  originalValue: { numerator: number; denominator: number }; // 원본 숫자 (예: 3/4)
  correctAnswer: { numerator: number; denominator: number }; // 정답인 역수 (예: 4/3)
  distractors: Array<{ numerator: number; denominator: number }>; // 오답 선택지 배열
}

interface QuizState {
  score: number;             // 현재 점수
  combo: number;             // 연속 정답 횟수 (피버 모드 적용 가능)
  timeRemaining: number;     // 남은 시간 (밀리초 단위)
  currentQuestionIndex: number; // 현재 문제 인덱스
  isGameOver: boolean;       // 게임 종료 여부
  masteryLevel: number;      // 유저의 평균 응답 속도에 따른 숙련도 지표
}
```

### 1-2. 점진적 과부하 AI 알고리즘 (Progressive Overload Logic) - **[피드백 반영]**
* **단순 크기 증가 지양**: 겉보기만 큰 숫자 위주로 바뀌는 것을 지양합니다.
* **숙련도 비례 복잡성 조정 (Dynamic Scaling)**: 
  * 유저가 평균 반응 속도 '1초' 미만의 임계점(Mastery Threshold)을 돌파하면 AI 컨트롤러가 즉시 개입합니다.
  * 단순히 숫자의 크기를 키우지 않고 **분모와 분자의 소수(Prime number) 비율 증가**, **가분수와 대분수를 섞어서 제시하는 패턴**, **약분이 필요한 형태** 등으로 문제의 '본질적 복잡도'를 동적으로 상향 조절합니다.
* **타이머 연동**: 연속 정답(`combo`) 달성 시 보너스 시간(+2초)을 부여하고, 오답 시 시간 페널티(-3초)를 주어 정확도를 요구합니다.

## 2. [중급] Bridge Builder: 물구나무 애니메이션 시스템
**목표**: "나눗셈은 곱셈으로, 뒤의 숫자는 역수로"라는 원리를 시각적으로 완벽하게 각인. 

### 2-1. 수학적 원리와 시각적 매핑 (Principle-First)
아이가 식을 수동으로 치환하는 것을 넘어, **중력이 반전되며 숫자가 물리적으로 뒤집히는 경험**을 전달합니다.

### 2-2. Framer Motion 기반 애니메이션 시퀀스
* **시퀀스 1 (기호 변환)**: 나눗셈 기호(`÷`)를 클릭하면 선을 향해 회전/병합되며 곱셈 기호(`×`)로 변환됩니다.
* **시퀀스 2 (물구나무 텀블링)**: 대상 분수 컨테이너 전체에 `rotateX: 180` 애니메이션을 발생시키고 90도 지점에서 데이터를 Swap하여 완벽한 역수 형태를 시각화합니다.

### 2-3. 'Why First' 로직 및 인터랙티브 인터벤션 (Interactive Intervention) - **[피드백 반영]**
* **오답 처리의 패러다임 전환**: 아이가 오답을 선택했을 때 단순 X표를 띄우지 않습니다.
* **'원리 증명' 모달 (Visual Proof Modal)**: 오답 시 화면이 일시 정지되며, "왜 역수를 곱해야 할까?"라는 질문과 함께 짧은 인터랙티브 모달이 등장합니다.
* **시각적 증명 게임화**: 모달 내에서 아이가 직접 원래 숫자 변의 박스와 역수 변의 박스를 드래그하여 포개면("곱하면"), 두 박스가 상쇄되며 **거대한 황금색 '1'**로 합쳐지는 애니메이션이 재생됩니다. 이를 통해 역수 관계의 본질(곱하면 1이 되는 수)을 뇌리에 강제로 각인시킵니다.

## 3. [고급] Ratio Detective: 스토리텔링 추리 미션
**목표**: 비례식과 역수가 실제 생활에서 사용되는 패턴(속력, 환율, 배분)을 사건 파일 형태로 제공하여 추리 형식으로 문제를 해결.

### 3-1. 실생활 시나리오 아키텍처 및 데이터 확장 - **[피드백 반영]**
기존 속력 도출 시나리오뿐만 아니라 '파이/피자 분할(배분)' 시나리오를 핵심 시각화 요소로 편입합니다.
```typescript
interface ScenarioMission {
  missionId: string;
  theme: 'speed' | 'currency' | 'pizza_sharing'; // 배분(Pizza Sharing) 추가
  context: string; 
  clues: string[]; // 파편화된 단서들
  relationFormula: 'inverse_proportion' | 'direct_proportion';
  hiddenValue: number; 
}
```

### 3-2. 전개 및 해결 로직 (UI Flow & Visualization)
1. **사건 파일 브리핑**: 속력, 환율, 혹은 인원수(피자 배분) 상황이 스크린에 타이핑됩니다.
2. **시각적 단서 배치 (피자 배분 연출)**:
   * "피자 배분(Pizza Sharing)" 테마의 경우, 우측에 인터랙티브 커스텀 피자 판 UI를 렌더링합니다.
   * 유저가 사람 수 단서 카드를 저울에 올릴 때마다 사람 수(N)가 증가하고, 그에 반비례하여 **피자 판이 1/N 크기의 조각으로 실시간 쪼개지는 파티클 그래픽 애니메이션**을 제공합니다.
3. **역수 원리 발동 (Validation)**: 사람 수(N) $\times$ 한 명이 먹는 양(1/N 조각) $= 1$(피자 한 판 전체) 로직이 수평저울 애니메이션과 결합되어 증명됩니다.
4. **정답 도출**: 수평이 맞춰지면 추리가 성공하며 핵심 원리 스탬프가 찍힙니다.

## 4. [특수 모듈] Gemini API 기반 실시간 음성(TTS/STT) 연동 
**목표**: 철수와의 상호작용을 극대화하여 집중력을 높이는 AI 퍼스널 튜터 구축.

### 4-1. 튜터 설정 아키텍처
* **보이스 커스터마이징 UI**: 상단 네비게이션에 `설정` 버튼을 두어, 30대 남/녀의 가장 자연스러운 한국어 목소리를 선택할 수 있는 환경설정 메뉴를 구성. (Gemini Multimodal 프로토콜 상정)

### 4-2. TTS (Text-to-Speech) 마법사 구현 방안
* 스텝 전환이나 오답/정답 판정 시, `useEffect` 훅에서 트리거를 받아 동적으로 프롬프트를 렌더링.
* "철수야, 너무 잘했어요! 이번엔 숫자를 물구나무 세워볼까요?"와 같이 파라미터화된 텍스트 스크립트를 서버사이드(혹은 브라우저 Native) 기반으로 읽어주는 오디오 파이프라인 개설.

### 4-3. STT (Speech-to-Text) 보이스 컨트롤 방안
* 철수의 음성("오분의 삼이요!")을 실시간으로 감지하여 분수(`3/5`) 데이터 타입으로 변환하는 NLP 정규화 로직(Normalizer) 모듈화.
