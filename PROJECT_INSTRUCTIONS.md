# [프로젝트 지시서 및 수행 지침] 화이트라벨 에듀테크 프레임워크 (구 Flip & Match)

## Ⅰ. 프로젝트 개요
* **프로젝트명**: 개별 맞춤형 에듀테크 자동화 프레임워크 (철수 케이스 포함)
* **총감독**: 허대표
* **조감독 및 번역가**: 수풀리 (Gemini)
* **개발 실행사**: AG (Antigravity)
* **핵심 목표**: 단 1개의 앱을 만드는 것을 넘어, `config.json` 설정 파일만으로 1시간 이내에 특정 학습자(예: 철수)와 과목(수학, 영어 등)에 최적화된 맞춤형 학습 앱을 봇으로 찍어내는 **'레고 블록식 범용 화이트라벨(White-Label) 구조'**를 확립하는 것.

## Ⅱ. 4대 규격화 지침 (Core Principles)
1. **학습자 프로필 기반 동적 브랜딩 (Dynamic Branding)**: `config.json` 파일 하나만 수정하면 앱 전체의 타이틀(예: 철수의 숫자 마법사), 색상 테마, 텍스트가 자동으로 치환되도록 테마 엔진을 규격화할 것.
2. **표준 학습 모듈 레고화 (Modular UI)**: 뒤집기, 연결하기, 추리하기 등의 UI 패턴을 공용 컴포넌트로 선언. 수학뿐 아니라 영어(단어 뒤집기 등)에도 즉시 적용 가능하게 과목 종속성을 배제할 것.
3. **AI 기반 콘텐츠 자동 생성 라인 (AI Content Pipeline)**: Claude 3.5 Sonnet을 활용하여 난이도별 연습 문제를 무한 생성하는 자동화 스크립트를 도입하고, 맞춤형 피드백 시스템을 표준화할 것.
4. **멀티모달 음성 표준 인터페이스 (Standard Voice UI)**: 30대 남녀 음성을 지원하는 공통 '표준 보이스 라이브러리' 구비. 학습자의 이름을 동적 삽입(TTS) 및 음성 인식(STT) 처리 모듈화.

## Ⅲ. 개발 상세 지시서 (Technical Specification)
* **[1단계] 매칭 & 뒤집기 레고 모듈 (Flip UI)**: 스피드 정답 찾기. (수학-역수, 영어-동의어/반의어 등 범용 데이터)
* **[2단계] 변환 & 텀블링 레고 모듈 (Bridge/Tumbling UI)**: 기호/의미를 변환하며 애니메이션 실행.
* **[3단계] 상황 추리 레고 모듈 (Detective Sandbox)**: 텍스트 시나리오 단서 조합 및 저울 애니메이션.

## Ⅳ. 작업 모드 가이드라인 (AG Mode Strategy)
* **Planning Mode (기획)**: 자동화 워크플로우 보고서 작성, 공용 컴포넌트 아키텍처, 프레임워크 구조 기획 시 사용.
* **Fast Mode (실행)**: JSON 기반의 단순 교체 작업, 마운트 및 배포 스크립트 실행.

## Ⅴ. 행정 사항 (자동화 규정)
* AG는 신규 의뢰 진입 시 **'단 1시간 이내'**에 학습자 맞춤형 앱의 뼈대를 100% 자동 생성할 수 있는 **[자동화 워크플로우 보고서]**를 제출할 것.
* 모든 코드는 재사용성을 극대화하여 특정 과목 단위의 하드코딩을 배제함.
* 시각 자료 확정 및 배포는 허대표 총감독님의 최종 승인 하에 진행함.
