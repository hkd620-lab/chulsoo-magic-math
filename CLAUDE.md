## 2. 허대표 총감독님 핵심 철학 (Core Principles) 및 최종 구현 현황

### Ⅰ. 원리 우선주의 (Why before How)
- **실생활 시각화 도입 완료**: 
  - 분수: 피자를 나누는 따뜻한 일러스트 적용.
  - 역수: 1/2로 줄어드는 레이싱카 스피드와 시간 시각화 적용.
  - 통분: 공통 기준을 맞추는 달러-원화 환율 교환 시각화 적용.
- **인터랙티브 애니메이션 적용 완수**: 
  - [1단계: 뒤집기 대장] 중앙 타겟 카드를 터치 시 `Framer Motion` 물리 엔진을 통해 3D로 180도 덤블링하는 UI 구축.
  - [2단계: 마법 다리] 나눗셈(÷)이 곱셈(×)으로 치환될 때 뒷 숫자가 동적으로 뒤집히며 분수로 변환되는 UI 구축.

### Ⅱ. 완전 학습 시스템 (3-Step Mastery)
- **강제 연습 관문 (Gate System)**: 개념 설명 직후, 직접 클릭해서 맞히는 **3연속 미니 퀴즈**를 통과해야만 다음 챕터의 자물쇠가 풀리는 잠금 해제 시스템 구현.
- **20문항 실생활 연습 확장**: [3단계: 비밀 탐정] 레시피, 속력, 환율 데이터 기반 20가지 최고 난이도 미션 뱅크 구현 완료.
- **주기적 마법 보상**: [도전! 마법 퀴즈] 5문제 간격으로 성취도를 자극하는 '마법 팝업(왕관 및 칭찬 보이스)'이 폭발하도록 트리거 적용.

### Ⅲ. 사용자 경험 (UX) 최적화
- **온디맨드 오디오 정밀 배치**: 상단 바 자동재생을 제거하고, 모든 개념 설명, 스토리텔링 문구, 문제 요소마다 1:1 매칭되는 **<스피커 아이콘(InlineSpeaker)>** 전수 배치 완료.
- **Bottom Navigation 최적화**: 화면 하단 영역으로 메뉴를 완전히 분리하여 콘텐츠 중첩 및 렌더링 겹침 현상 원천 차단.
- **용어 클렌징**: '메타인지', '몬스터 포획' 등의 추상적 단어를 전부 걷어내고 철수의 눈높이에 맞춘 쉽고 다정한 용어(예: 도전! 마법 퀴즈, 비밀 탐정 사무소)로 100% 매핑 완료.

## 3. Git 커밋 및 Hub 운영 정책 (Git & Hub Policy) 🛠️

프로젝트의 이력 관리와 협업 효율을 위해 다음의 엄격한 규칙을 적용한다.

### Ⅰ. 커밋 메시지 형식 (Commit Convention)
- **형식**: `[타입] 제목 — 구체적인 변경 내용 요약`
- **타입 종류**: 
    - `[기능추가]`: 새로운 마법(기능)을 만들었을 때
    - `[버그수정]`: 오류(숫자 실종, JSX 삼항 연산자 오류 등)를 고쳤을 때
    - `[디자인]`: 레이아웃, 버튼 간격, 이미지, Framer 애니메이션 수정 시
    - `[데이터]`: 실생활 문제 20문항 추가 및 수학 로직 수정 시
- **필수 포함**: 무엇을 왜 변경했는지, 어떤 파일이 영향받았는지 **한국어로 자세히** 기록할 것.

### Ⅱ. GitHub(Hub) 관리 수칙
- **보안 최우선**: `.env` (TTS API Key 등) 파일은 절대 Push하지 않으며, `.gitignore` 필수 유지.
- **수정사항 검수**: 렌더링 체킹 및 500 에러 유무 등을 로컬에서 반드시 1차 검수 후 커밋.

## 4. 작업 프로세스 및 R&R 가이드라인 (효율 극대화) 🪙

허대표 총감독님과 AG 시스템의 협업 규칙은 다음과 같다.

- **총감독 (USER)**: 기획 의도 및 최종 의사 결정 통보.
- **AG (AI Agent)**: 지시된 모든 기술 구현(React, 애니메이션, 이미지 애셋 등)을 사용자 코딩 개입 없이 독립적으로 수행.
- **토큰 절약 및 소통**: AG는 총감독에게 시스템 구현 세부 코드를 장황하게 설명하지 않으며, **"무엇을(수정/추가 사항), 왜(목적 달성 확인), 어떻게(해결 방식)"**의 3단계로만 압축하여 핵심을 보고한다.
